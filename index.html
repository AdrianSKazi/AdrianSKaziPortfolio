Finance and Econometrics

Warsaw School of Economics QMEIS - Fraud Rank App
21-07-2022

In Application I took 3 types of frauds which are Money Mule, Credit Card Credential Theft and Authorized Push Payments. To detect each crime I used 3 self-created variables which are

  • sameInOut - sum and count of transit transactions, so count of pairs of transactions where one is incoming and one outgoing and both are in the same amount
  • atm - sum and count of cash withdrawal transactions
  • newRecipients - sum and count of transactions to new recipients in a given month
After creating variables I used Facebook Prophet model for anomaly detection among variables for each customer in each month. After that I created Fraud Rank, which is simply descending rankng of absolute error between empirical value, detected as anomaly and predicted value defined by Prophet. Model is about to serve AML analyst for prioritizing cases not for rejecting any case by defining any customer as not suspicious. Dataset used for project is real data coming from Czech bank, data was cleared from any personal data, for more information please refer to dataset website.

Asseco's Codeplay Hackathon - Fraud detector
24-10-2020

Fraud detector was part of Asseco Codeplay 2020 Hackathon. With our team we developed 3 steps fraud detection system, which includes logistic regression trained on already labeled data and provided by bank, KPMG's demographic-psychologic profiling of usual fraudsters and Benford's law run on transaction numbers of each customer. After model run each customer was about to receive 1, 2 or 3 alerts. Model wasn't created for excluding customers from circle of suspects, instead it was rather a prioritization system for AML analysts to handle most suspicious customer first. Project was created in a duration of 4 hours and pushed our team to the final 10 out of more than 100 teams, however code in not finalized and debugged. Ideas from this event are being used for further development on different projects.
